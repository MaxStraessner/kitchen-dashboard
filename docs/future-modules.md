# Future modules

Recommended sequence after acceptance of the foundation:

1. Deployment on Hostinger with isolation, HTTPS, backups, and rollback
2. User accounts and secure authentication
3. Households, membership, roles, and permissions
4. Mobile family interface
5. Shared task persistence and workflows
6. Realtime synchronization as an optional transport
7. Display-device pairing and remote display state
8. Bring integration
9. Spotify integration
10. Calendar source management and authenticated providers
11. Smart-home read/control adapters
12. Camera views with explicit privacy controls
13. N8N automations

N8N is an optional integration edge, not a core runtime dependency. Each new capability should preserve provider isolation, versioned contracts, household authorization, and graceful partial failure.
